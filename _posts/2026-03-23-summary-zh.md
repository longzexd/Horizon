## [Starlette 1.0 发布，作者探讨其意义与 LLM 集成](https://simonwillison.net/2026/Mar/22/starlette/#atom-everything) ⭐️ 8.0/10

作为基础 Python ASGI 网络框架的 Starlette 发布了 1.0 版本，标志着其首个稳定的主要版本。作者 Simon Willison 探讨了这一里程碑事件，并尝试使用 Claude AI 的 skill 功能来生成兼容新版本破坏性变更的代码，特别是从 `on_startup`/`on_shutdown` 参数转向新的 `lifespan` 异步上下文管理器。 此次发布意义重大，因为 Starlette 是广受欢迎的 FastAPI 框架的基础，这意味着其稳定性和演进直接影响着现代 Python 网络生态的很大一部分。探索针对新 API 的 AI 辅助代码生成，突显了保持 LLM 输出与不断演进的库接口同步所面临的实际挑战和机遇。 Starlette 1.0 中最值得注意的破坏性变更是用新的 `lifespan` 参数取代了 `on_startup` 和 `on_shutdown` 事件处理器，该参数使用异步上下文管理器，为管理应用生命周期提供了更清晰、更结构化的方式。作者指出，Starlette 的简洁性（类似于 Flask）使得 LLM 特别容易根据提示生成功能性的应用程序。
