## [逆向工程揭示如何绕过 Claude Code 的 Bun 运行时签名](https://a10k.co/b/reverse-engineering-claude-code-cch.html) ⭐️ 7.0/10

一项技术分析成功逆向工程了 Claude Code 的 API 认证机制，特别是其专有的 `cch` 头部和 `cc_version` 后缀。该研究展示了如何在不依赖其捆绑的 Bun 运行时的情况下伪造这些签名，并通过一个 Python 概念验证实现了对快速模式等受限功能的访问。 这很重要，因为它暴露了 Claude Code 在功能门控和用量归因方面的一个潜在弱点，揭示了其签名机制更多是关于计费归因和功能控制，而非强健的安全防护。这凸显了依赖混淆的客户端逻辑进行访问控制的风险，并可能影响其他类似 AI 编程工具设计其认证和授权系统的方式。 `cch` 值由 Bun 运行时使用 xxHash64 算法对整个 JSON 请求体计算得出，而 `cc_version` 的 3 位后缀则是由首条用户消息中的指定字符、内置盐值和版本号经 SHA-256 哈希运算得出。分析结论认为，该机制并非强访问控制，而是服务于计费归因和功能门控的目的。
