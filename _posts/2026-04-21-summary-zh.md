## [Claude Desktop 在 Chromium 浏览器中静默注册原生消息传递清单](https://www.thatprivacyguy.com/blog/anthropic-spyware) ⭐️ 7.0/10

Anthropic 的 Claude Desktop 应用在 macOS 上安装后，会在七个基于 Chromium 的浏览器目录的 NativeMessagingHosts 文件夹中自动创建原生消息传递清单文件，且未经用户同意或告知。该清单文件指向 Claude 应用内的一个二进制文件，并预授权三个特定的 Chrome 扩展 ID 与该主机通信，从而启用了浏览器自动化能力。 这一行为之所以重要，是因为它在桌面 AI 应用和网页浏览器之间建立了一个隐藏的通信桥梁，使其可能访问浏览器标签页、登录状态和网页内容。这引发了严重的隐私和安全担忧，因为它未经用户知情、缺乏文档说明，并且可能被恶意扩展利用以获得未经授权的系统访问权限。 该清单文件在应用启动时创建，并在后续每次运行时重写，即使对应的浏览器并未安装也会创建相关目录。Anthropic 的公开文档未提及桌面应用的此桥接功能，仅 Claude Code 有单独说明，而该桥接启用的能力包括打开浏览器标签页、共享登录状态、读取 DOM 以及填写表单。

**背景**: 原生消息传递（Native Messaging）是 Chrome/Chromium 扩展的一个 API，允许扩展与用户计算机上安装的原生应用程序进行通信。为实现此功能，原生应用必须在特定目录（如 NativeMessagingHosts）中注册一个清单文件，该文件定义了主机可执行文件的路径，并列出了获授权可与其连接的扩展 ID。此机制通常用于密码管理器或自动化工具等合法目的，但需要在扩展安装时获得用户同意。
